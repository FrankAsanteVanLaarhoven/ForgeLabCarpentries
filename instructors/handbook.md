# Instructor Handbook

Practical, day-of guidance for running a ForgeLab workshop. Pairs with the cross-episode
[Instructor Notes](../lessons/instructor-notes.qmd) on the lesson site.

## 1–2 weeks before

- [ ] Confirm co-instructor(s) + helper(s). **Never teach alone.**
- [ ] Assign roles (lead, chat/Q&A monitor, tech/breakout support).
- [ ] Create the workshop folder from `workshops/_template/` and fill it in.
- [ ] Create the collaborative-notes doc; test it.
- [ ] Send the pre-survey and setup instructions to registrants.
- [ ] Offer an optional "install party" / setup help session.
- [ ] Test the cloud fallback (Codespace/Docker) yourself.

## Day of — before learners arrive

- [ ] Test audio, screen share, font size, and the high-contrast theme.
- [ ] Open: collaborative notes, lesson site, reference/cheat sheet, chat channel.
- [ ] Enable captions (online). Confirm a human will watch for caption errors.
- [ ] Helpers know their roles and the reporting path for CoC issues.

## Running the room

- Welcome + **introduce the Code of Conduct explicitly** (not just a link).
- Icebreaker; set the tone: mistakes are expected and welcome.
- Run the loop: **short demo → challenge → debrief.** Type, narrate, don't paste.
- Use **formative feedback** constantly:
  - In person: red/green sticky notes (red = stuck). Don't advance while reds are up.
  - Online: emoji reactions / polls; helpers triage the chat.
- **Equity in attention:** rotate who you call on; watch quiet learners and breakouts.
- Take **frequent breaks**. Online, more and shorter.

## Handling common situations

| Situation | Response |
|-----------|----------|
| A learner's setup is broken | Move them to the cloud fallback; a helper assists 1:1. |
| The room is moving at very different speeds | Pair fast + struggling learners; give stretch goals. |
| You hit a live bug you can't fix fast | Narrate your debugging, then use a prepared checkpoint. |
| A CoC concern is raised | Follow the [CoC](../CODE_OF_CONDUCT.md) reporting path; take it seriously and discreetly. |
| Energy is flagging | Break early; switch to a hands-on challenge or breakout. |

## Wrap-up

- Summarize key points; show the path to the next track/capstone.
- Collect end-of-day feedback (one thing that worked / one to improve).
- Send the **post-survey immediately**.

## After

- Schedule the 3–6 month follow-up survey.
- Run a [retrospective](../community/retrospective-template.md) with the teaching team.
- File issues/PRs for any lesson rough edges. Update the [dashboard](dashboard.md).
