# Track: Capstones & Certification

**Audience:** learners who have completed one or more tracks and want to demonstrate mastery.
**Prerequisites:** at least one specialization track.
**Outcome:** a portfolio-grade project and a ForgeLab micro-credential.

## How capstones work

A capstone is an end-to-end project that integrates skills across tracks. Learners propose,
build, and present; a mentor and peers review against a rubric. Success earns a **badge**.

### Process

1. **Propose** — submit a one-page plan (problem, approach, success criteria, scope) as an issue.
2. **Build** — work in a Git repo with CI; weekly check-ins with a mentor.
3. **Review** — peer + mentor review against the rubric below; iterate.
4. **Present** — short demo + reflection (what worked, what failed, what you'd change).
5. **Credential** — issue a badge recording the track(s), date, and project link.

## Example capstones

| Capstone | Integrates |
|----------|------------|
| Language-commanded navigation in sim | Robotics + AI (VLN) + Foundations |
| Containerized perception service with CI/CD and rollback | Cloud + Robotics + Foundations |
| World-model planner with honest evaluation report | AI + Foundations |
| Provenance-anchored data pipeline for a robot fleet | Blockchain + Cloud + Foundations |

## Rubric (100 pts)

| Criterion | Pts | What we look for |
|-----------|-----|------------------|
| Correctness & function | 25 | It works and does what was proposed |
| Reproducibility | 20 | Anyone can clone, set up, and run it; CI green |
| Engineering quality | 15 | Clear structure, version control hygiene, tests |
| Honest evaluation | 20 | Real metrics, failure analysis, limits stated |
| Communication | 20 | Clear README + demo; reflection on lessons learned |

## Micro-credentials

Badges are issued per completed track and per capstone. Each badge records: recipient, track,
issue date, and a link to the verifiable project. Track instructor checkout and learner badges
in the [instructor dashboard](../../instructors/dashboard.md).

::: note
We weight **honest evaluation** heavily on purpose: reporting what *didn't* work — with metrics
— is a core ForgeLab value, mirroring how we evaluate our own workshops.
:::
