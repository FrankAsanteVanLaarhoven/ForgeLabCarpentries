# Instructor Training Curriculum

A pedagogy-first program adapted from the Carpentries Instructor Training for ForgeLab's
robotics/AI context. **Tools second, teaching first** — a great facilitator can teach any
episode; deep tool knowledge without facilitation skill cannot.

## Format

- **2 full days in-person**, *or*
- **4 × half-day online sessions** (highly effective; recommended for distributed cohorts).

Cohort size: 6–20 trainees with 1–2 trainers. Active throughout — trainees practice, not listen.

## Module map

| # | Module | Trainees can… |
|---|--------|---------------|
| 1 | How people learn | explain mental models, cognitive load, and why live coding works |
| 2 | Motivation & demotivation | create conditions for motivation; avoid common demotivators |
| 3 | Active learning & formative assessment | run demo→practice→debrief; use frequent checks |
| 4 | Inclusive teaching | apply concrete inclusion practices; reduce stereotype threat |
| 5 | Live-coding facilitation | type-and-narrate, recover from errors, pace to the room |
| 6 | Questioning & diverse learners | ask diagnostic questions; support fast and struggling learners together |
| 7 | Co-teaching dynamics | share the room; hand off; run the helper/chat roles |
| 8 | Online-specific skills | manage Zoom, chat moderation, breakouts, engagement, fatigue |
| 9 | Teaching demos | deliver short demos with peer + trainer feedback (repeated) |

## Online-specific skills (Module 8 detail)

- Clear instructor roles: **lead facilitator**, **chat/Q&A monitor**, **tech/breakout support**.
- Shorter demos + more frequent, shorter exercises to combat fatigue.
- Breakout rooms for pair programming and small-group challenges.
- Cloud fallbacks ready (Codespaces, Docker) for setup failures.
- Accessibility: captions (auto + human review), transcripts, clear audio, high-contrast slides.
- A backup plan for every common failure mode (audio, screen share, a learner's broken setup).

## Teaching demos (the core practice)

Each trainee delivers **multiple short demos** (5–10 min) and receives structured peer +
trainer feedback. Demos progress from "teach anything you know" to "teach a ForgeLab episode."
This is where skills actually form.

## After training

- Complete [checkout/certification](checkout-certification.md).
- Join the community of practice (regular meetings, observation swaps).
- Optional pathways: **Lesson Development Training** (author/maintain curricula) and
  **Train-the-Trainer** (scale locally).

## Trainer notes

- Model everything you teach — your training session *is* a live-coding workshop.
- Give feedback the way you want instructors to: specific, kind, actionable, balanced.
- Track each trainee's progress in the [dashboard](dashboard.md).
